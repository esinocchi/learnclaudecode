# Peer Review Revision Task List

---

## Front Matter Revisions

### Table of Contents
- [ ] Add clear, descriptive titles for each resource in the TOC
- [ ] Add working hyperlinks for every TOC entry
- [ ] Expand TOC to include at least two levels (sections + subsections)

### Content & Scope Description
- [ ] Rewrite the intro/overview to read like a guide, not an essay — use bullets, short paragraphs, scannable formatting
- [ ] Explicitly state the content scope (what is and isn't covered)

### Audience & Purpose
- [ ] Add a clear audience statement near the top (e.g., "This guide is for Penn State freshman CS students...")
- [ ] Make the purpose statement more action-oriented and practical

### Assumptions
- [ ] Add a "Prerequisites" or "What You Should Already Know" section that explicitly states assumed knowledge (basic coding, terminal use)

### Organization Explanation
- [ ] Add a paragraph or visual in the front matter that explains the guide's structure (learning the tool → building → showcasing)
- [ ] Make the progression logic explicit so readers know why it's ordered that way

### Tips for Use
- [ ] Add a dedicated **"How to Use This Guide"** section that includes:
  - [ ] Where to start
  - [ ] What order to follow
  - [ ] How to use each resource entry

### Visual Separation
- [ ] Add consistent headings to separate front matter sections
- [ ] Use spacing and/or short labels to visually distinguish sections

---

## Abstract Revisions

### Links
- [ ] Add a direct, working URL to every single abstract entry — this is the top priority fix

### Consistency
- [ ] Standardize every abstract to follow this exact format:
  1. Title
  2. Link (URL)
  3. What it is
  4. Why it's relevant to the audience/purpose
  5. Tips for using it efficiently

### Completeness of the Four Questions
- [ ] Audit every abstract and confirm it answers:
  - [ ] What is this?
  - [ ] What's in it?
  - [ ] Why is it relevant to freshman CS students building with Claude Code?
  - [ ] How do you use it efficiently?
- [ ] Add or strengthen usage tips wherever they're weak or missing

### Length & Efficiency
- [ ] Cut repetitive or over-explained content from abstracts
- [ ] Rewrite long sentences to be shorter and more direct
- [ ] Ensure each abstract gets to the point in the first sentence (deductive style)

---

## Overall Guide Design & Style

### Usability / Web Design
- [ ] Ensure all links throughout the guide are clickable and functional
- [ ] Add clear section headings throughout the body of the guide
- [ ] Format the guide so it's scannable (not a wall of prose)

### Writing Style
- [ ] Audit every abstract for overly long sentences — trim them
- [ ] Make sure every abstract opens with a strong, specific topic sentence
- [ ] Confirm the writing style is deductive (main point first, details after)

### Step-by-Step Instructions
- [ ] Add actionable instructions where appropriate (e.g., "Click X, then read Y, then do Z")
- [ ] For each resource, consider adding a brief "Next Step" or "Start Here" prompt

---

> **Top 3 Priorities:** (1) Add links everywhere, (2) Standardize abstract format, (3) Shorten and tighten the writing.
